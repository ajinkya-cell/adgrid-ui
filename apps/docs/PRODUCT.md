# Product

## Register

product

## Users

Design engineers building dark-themed, premium interfaces — marketing sites, portfolios, landing pages, and product UI. They reach for void-ui when they want components that feel crafted, not templated.

## Product Purpose

A dark-first React component library that ships premium, production-grade animated components. Each component is a self-contained artifact — WebGL backgrounds, motion buttons, image treatments — that elevates any project it's dropped into. Success means developers reach for void-ui before Framer Motion or GSAP primitives because the components already embody the hard part: taste, motion design, and polished defaults.

## Brand Personality

Elegant, minimal, precise. The tone is quiet confidence — nothing shouts, nothing performs. Every component earns its place by doing one thing exceptionally well, with no superfluous decoration.

## Anti-references

Avoid the saturated AI-slop patterns: glassmorphism, gradient text, numbered section markers, tiny uppercase kickers, hero-metric templates, and side-stripe borders. No cyberpunk, no neon, no gimmicks. The library should feel like it was designed by a human with restraint, not generated by a model optimizing for "premium."

## Design Principles

1. **Dark-first, not dark-only.** Every component starts in dark mode; light mode is a deliberate adaptation, not an afterthought.
2. **Motion with purpose.** Every animation serves the user's understanding — never decoration. Staggered entrances, specular highlights, cursor responses are earned by what they communicate.
3. **Obsidian materials.** Surfaces should feel like physical materials — stone, metal, glass, embers — rendered with restraint. No glow for glow's sake.
4. **Self-contained artifacts.** Each component carries zero external style dependencies beyond its own code. Drop it in, it works.
5. **Reduce the reflex.** The first design instinct is usually the AI-default one. Pause, question it, choose the quieter, more precise alternative.

## Accessibility & Inclusion

Best-effort with practical focus: reduced-motion support via `prefers-reduced-motion`, keyboard-navigable interactive components, and sufficient color contrast. Full WCAG AA is aspirational but not blocking.
